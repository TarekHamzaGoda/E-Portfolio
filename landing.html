Forum Discussion

One important aspect of Object-Orientated Programming is code reusability. Over the years, software engineers were able to advance in OOP practices across all industries by leveraging existent codes. OOM or object-oriented metrics aims to measure the reusability of software codes and data organization that can assist with efficient OOP development. In an article by Padhy et al (2018), researchers have established several properties in which code quality is measured.

  • 1. “Architecture driven approach (ADP)”
    • 2. “An Algorithm used in the program (AP)” 3. “Modules in the program (MIP)” 4. “Requirement Analysis (RA)” 5. “Design Patterns (DP)” 6. “Documentation in project (DIP)” 7. “Test Cases/Test Design (TCTD)” 8. “Used in the data project (UD)” 9. “Models in the project (MP)” 10. “Knowledge requirement (KR)” 11. “Service contracts (SC)” List priority above is organized based on a personal opinion. Software architecture ADP and algorithm AP are the main structure of the system. Code reusability often depends on the success of both aspects. Thus, ADP and AP are one of the first indicator of code quality. MIP, RA, and DP are tested with respect to project requirements. Similarities in implementation can also indicate the reusability of the code. Another important code quality indictor is TCTD, which in return insure the validity of the pervious reusable properties.
